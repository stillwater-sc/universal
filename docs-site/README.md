# Universal Numbers Documentation Site

Starlight-based documentation site for the Universal Numbers Library, deployed to GitHub Pages at https://stillwater-sc.github.io/universal/.

## Architecture: 100% Generated Content

`src/content/docs/` is entirely generated by `sync-content.mjs`. **Never edit files there** -- your changes will be lost on the next build.

```
SOURCES (committed, edit here)              GENERATED (gitignored)
──────────────────────────────              ──────────────────────
docs/*.md (tutorials, design, etc)    ──┐
docs/number-systems/*.md               │
docs/ucalc/*.md                        ├─► sync-content.mjs ──► src/content/docs/
docs/site/*.md, index.mdx              │   (wipes + regenerates on every build)
repo root (PURPOSE.md, etc)           ──┘
```

## Three Source Locations

| Location | What goes here | Processing |
|----------|---------------|------------|
| `docs/` | Plain markdown (number systems, tutorials, design, ucalc) | Frontmatter prepended, links rewritten to clean URLs |
| `docs/site/` | Starlight-native pages (MDX landing page, section indexes, getting-started guides) | Copied verbatim (already have YAML/MDX frontmatter) |
| Repo root | `PURPOSE.md`, `CONTRIBUTORS.md`, `CODE-OF-CONDUCT.md`, `CHANGELOG.md` | Frontmatter prepended, links rewritten |

### docs/ files (FILE_MAP)

Plain markdown with `# H1` headings. The sync script extracts the heading as the YAML `title:`, rewrites image paths and `.md` links to Starlight clean URLs, and writes the result.

- `docs/number-systems/*.md` -- 32 number system docs
- `docs/ucalc/*.md` -- ucalc overview, examples, step-by-step, MCP server
- `docs/tutorials/*.md` -- tutorials (type parameterization, posit refinement, etc.)
- `docs/command-line-tools.md`, `docs/mixed-precision-*.md`, `docs/block-formats.md`
- `docs/design/*.md`, `docs/RELEASE_PROCESS.md`
- Various top-level docs (`cross-compilation.md`, `troubleshooting.md`, etc.)

### docs/site/ files (SITE_FILES)

Starlight-native pages that already have YAML frontmatter (or MDX with component imports). Copied verbatim without modification.

- `docs/site/index.mdx` -- splash landing page with hero section and Card components
- `docs/site/getting-started/` -- `index.md`, `installation.md`, `docker.md`, `first-program.md`
- `docs/site/tutorials/index.md`, `docs/site/build/index.md`, `docs/site/contributing/index.md`
- `docs/site/design/index.md`, `docs/site/exact-arithmetic/index.md`
- `docs/site/resources/citation.md`, `docs/site/resources/presentations.md`

## What sync-content.mjs Does

Runs automatically before `npm run dev` and `npm run build`:

1. **Wipes** the entire `src/content/docs/` directory
2. Processes `FILE_MAP` entries: reads each `docs/*.md`, extracts `# H1` as YAML title, rewrites image paths and `.md` links, writes to destination
3. Processes `ROOT_FILE_MAP` entries: same as above for repo-root files
4. Copies `SITE_FILES` entries: verbatim from `docs/site/` to destination
5. Copies static assets (`docs/img/`, `docs/closure_plots/`, `docs/presentations/`) to `public/`
6. Clears the Astro cache to ensure changes are detected

## Common Workflows

### Edit an existing number system doc

```bash
vim docs/number-systems/dfloat.md
cd docs-site && npm run build && npm run preview
```

### Add a new number system doc

```bash
# 1. Create the source file
vim docs/number-systems/newtype.md

# 2. Register in sync-content.mjs FILE_MAP
#    Add: 'number-systems/newtype.md': 'number-systems/newtype.md',

# 3. Add to sidebar in astro.config.mjs

# 4. Build and preview
cd docs-site && npm run build && npm run preview
```

### Edit a Starlight page (getting-started, landing page, section indexes)

```bash
vim docs/site/getting-started/installation.md
cd docs-site && npm run build && npm run preview
```

### Edit repo-root docs (PURPOSE, CHANGELOG, etc.)

```bash
vim PURPOSE.md
cd docs-site && npm run build
```

### Clean and rebuild from scratch

```bash
cd docs-site
npm run clean    # removes src/content/docs/, dist/, public/img/, .astro/
npm run build    # regenerates everything
```

## Gotchas

- **Never edit `src/content/docs/`** -- it's wiped on every build. Edit `docs/` or `docs/site/` instead.
- **Underscore-to-hyphen renames**: Three files are renamed during sync:
  - `dd_cascade.md` -> `dd-cascade.md` (URL: `/number-systems/dd-cascade/`)
  - `td_cascade.md` -> `td-cascade.md`
  - `qd_cascade.md` -> `qd-cascade.md`
- **Links in source markdown**: Use relative `.md` links (e.g., `[integer](integer.md)`). The sync script rewrites them to Starlight clean URLs. This keeps links working on GitHub too.
- **Sidebar configuration**: Number Systems sidebar items are manually configured in `astro.config.mjs` for grouping. Other sections use `autogenerate: { directory: '...' }`.
- **New SITE_FILES entries**: When adding a new Starlight-native page, add it to both `SITE_FILES` in `sync-content.mjs` and the sidebar in `astro.config.mjs`.

## Development

```bash
npm install        # install dependencies
npm run dev        # sync + start dev server with hot reload
npm run build      # sync + production build
npm run preview    # serve the production build locally
npm run sync       # run sync-content.mjs only (no build)
npm run clean      # remove all generated artifacts
```
