# docs-site Content Workflow

The documentation site uses [Astro Starlight](https://starlight.astro.build/).
Content is authored as Markdown in the repo's `docs/` directory and synced
into `docs-site/src/content/docs/` at build time by `sync-content.mjs`.

## Source of Truth

| Content | Source location | Synced to |
|---------|----------------|-----------|
| Number systems | `docs/number-systems/*.md` | `src/content/docs/number-systems/` |
| Tutorials | `docs/tutorials/*.md` | `src/content/docs/tutorials/` |
| Design docs | `docs/design/*.md` and `docs/*.md` | `src/content/docs/design/` |
| Mixed precision | `docs/mixed-precision-*.md` | `src/content/docs/mixed-precision/` |
| Build guides | `docs/cross-compilation.md`, etc. | `src/content/docs/build/` |
| Getting started | `docs/command-line-tools.md` | `src/content/docs/getting-started/` |

The mapping is defined in `FILE_MAP` inside `docs-site/sync-content.mjs`.

**Hand-written pages** (not synced) live directly in `src/content/docs/`:
- `index.mdx` -- site landing page
- `*/index.md` -- section index pages
- `getting-started/installation.md`, `docker.md`, `first-program.md`
- `resources/citation.md`, `presentations.md`

## Adding a New Document

1. **Create the source file** in `docs/` (or a subdirectory like `docs/tutorials/`):

   ```bash
   vim docs/tutorials/my-tutorial.md
   ```

   Use an H1 heading as the first line -- the sync script extracts it as the
   page title and generates Starlight frontmatter automatically:

   ```markdown
   # My Tutorial Title

   Content starts here...
   ```

2. **Add a FILE_MAP entry** in `docs-site/sync-content.mjs`:

   ```javascript
   'tutorials/my-tutorial.md': 'tutorials/my-tutorial.md',
   ```

   The key is the path relative to `docs/`, the value is the path relative
   to `src/content/docs/`.

3. **Update the section index** (if needed) in the hand-written
   `src/content/docs/tutorials/index.md`:

   ```markdown
   - [My Tutorial](../tutorials/my-tutorial/) -- description
   ```

4. **Verify** by running the sync and checking the output:

   ```bash
   cd docs-site
   npm run sync    # generates src/content/docs/tutorials/my-tutorial.md
   npm run dev     # preview at localhost:4321
   ```

## Commands

```bash
cd docs-site

npm run sync      # sync docs/ -> src/content/docs/ (generates pages)
npm run clean     # remove all generated files from src/content/docs/
npm run dev       # sync + start dev server
npm run build     # sync + build static site to dist/
npm run preview   # serve the built dist/ directory
```

## Rules

- **Never edit files in `src/content/docs/`** that are generated by sync.
  Edit the source in `docs/` instead. The `.gitignore` prevents accidentally
  committing generated files.

- **Never commit generated files.** If `git status` shows files under
  `src/content/docs/` that shouldn't be there, run `npm run clean`.

- **H1 heading required.** Source files must start with `# Title` (no YAML
  frontmatter). The sync script generates the Starlight frontmatter.

- **Hand-written pages use frontmatter.** Pages authored directly in
  `src/content/docs/` (like `index.md`) use standard Starlight YAML
  frontmatter since they bypass the sync pipeline.

- **Images** go in `docs/img/` and are copied to `public/img/` by the sync
  script. Reference them as `/universal/img/filename.png` in Markdown.
